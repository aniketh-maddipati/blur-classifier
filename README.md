# blur-classifier

This project fine-tunes **Qwen/Qwen3.6-35B-A3B** on [Tinker](https://github.com/thinking-machines-lab/tinker-cookbook) to classify photos into three blur categories: `intentional_blur` (deliberate artistic blur such as motion trails or shallow depth-of-field), `unintentional_blur` (camera shake, missed focus, or subject movement), and `sharp` (no significant blur). The goal is to build a labeled training set from real iPhone RAW/JPEG exports, establish a zero-shot baseline, fine-tune with a learning-rate sweep, and compare checkpoint accuracy against the baseline.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # then open .env and paste your Tinker API key
```

Add labeled photos to the dataset folders — one subfolder per class:

```
dataset/train/intentional_blur/   dataset/holdout/intentional_blur/
dataset/train/unintentional_blur/ dataset/holdout/unintentional_blur/
dataset/train/sharp/              dataset/holdout/sharp/
```

JPEG, PNG, TIFF, and RAW formats (CR2, NEF, ARW) are all supported.

The training launcher expects a manifest-first local dataset layout generated by `split_dataset.py`:

```text
dataset/
  split_manifest.csv
  train/{intentional_blur,unintentional_blur,sharp}/*
  holdout/{intentional_blur,unintentional_blur,sharp}/*
```

`train_blur.py` reads `dataset/split_manifest.csv` first and fails loudly if the manifest and on-disk folders drift apart. The holdout split is treated as the recipe's `test` split.

## Run order

```bash
python classify.py        # fast sanity check — pure assertions, no network call
python baseline_eval.py   # zero-shot accuracy on holdout set → results/baseline_results.csv
python train_blur.py experiment_dir=./vlm_classifier examples_per_class=5
python train_blur.py experiment_dir=./vlm_classifier
python eval_blur.py model_path=<checkpoint_or_tinker_path>
```

## Local blur training

`train_blur.py` mirrors the cookbook recipe launcher but uses the repo-local manifest-backed dataset builder instead of the recipe's HuggingFace dataset registry.

Defaults:

```text
renderer_name=qwen3_5_disable_thinking
model_name=Qwen/Qwen3.6-35B-A3B
lora_rank=32
batch_size=8
max_image_size=1024
max_length=8192
```

Recommended first run:

```bash
python train_blur.py experiment_dir=./vlm_classifier examples_per_class=5
```

Use that 5-shot run as a cheap pipeline check before a full training run. It validates the manifest, tokenizer/renderer pairing, dataset rendering, checkpoint writing, and evaluation wiring before you spend real money.

Full run:

```bash
python train_blur.py experiment_dir=./vlm_classifier wandb_project=vlm-classifier
```

Useful overrides:

```bash
python train_blur.py experiment_dir=./vlm_classifier max_image_size=480
python train_blur.py experiment_dir=./vlm_classifier max_image_size=1344
python train_blur.py experiment_dir=./vlm_classifier manifest_path=/absolute/path/to/split_manifest.csv
```

## Holdout evaluation

Run sampling evaluation against the local holdout split:

```bash
python eval_blur.py model_path=<checkpoint_or_tinker_path>
```

The evaluator prints:

- overall accuracy
- per-class precision and recall
- a 3x3 confusion matrix over `intentional blur`, `unintentional blur`, and `sharp`

Example with explicit overrides:

```bash
python eval_blur.py \
  model_path=<checkpoint_or_tinker_path> \
  model_name=Qwen/Qwen3.6-35B-A3B \
  renderer_name=qwen3_5_disable_thinking \
  max_image_size=1024
```
